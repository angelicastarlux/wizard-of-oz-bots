================================================================================
                    WHEN TO USE WHAT
================================================================================

‚úÖ FOR PRODUCTION (Users using your bot 24/7):
   1. Make sure Fly.io bot is running: flyctl scale count 1 -a wizard-of-oz-bots
   2. DO NOT run "node wizard-of-oz.js" locally
   3. Your bot is online 24/7 for everyone!

‚úÖ FOR TESTING LOCALLY (Making changes & testing):
   1. Stop Fly.io bot: flyctl scale count 0 -a wizard-of-oz-bots
   2. Run locally: node wizard-of-oz.js
   3. Test your changes
   4. Stop local bot: Press Ctrl+C in terminal
   5. Deploy to Fly.io: flyctl deploy -a wizard-of-oz-bots
   6. Start Fly.io bot: flyctl scale count 1 -a wizard-of-oz-bots

================================================================================
                    WHY CAN'T I RUN BOTH AT ONCE?
================================================================================

‚ùå LOCAL BOT (Terminal/VSC):
   - Runs on YOUR computer
   - Uses your Discord bot token
   - Responds to commands

‚ùå FLY.IO BOT (Cloud):
   - Runs on Fly.io servers (24/7)
   - Uses the SAME Discord bot token
   - Responds to commands

‚ö†Ô∏è  PROBLEM: Both bots have the same token, so Discord sees them as the SAME bot

‚úÖ SOLUTION: Only run ONE at a time!
